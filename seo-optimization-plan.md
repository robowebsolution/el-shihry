# خطة SEO + AEO + GEO متكاملة لموقع El Shihry قبل الإطلاق

## الملخص
- نعتمد الموقع الحالي كنقطة البداية، لكن نغيّر بنية اللغة من `cookie-based` إلى مسارات مستقلة: `/ar` و`/en`. هذا قرار أساسي لأن Google ما زال يوصي بعناوين منفصلة للغات بدل الصفحات المتكيّفة حسب اللغة/الكوكيز.
- الهدف التجاري الأول هو `qualified leads` عبر `WhatsApp` والمكالمات، لذلك كل بنية المحتوى والقياس والـ CTAs ستخدم هذا الهدف قبل أي توسع تحريري.
- الخطة تغطي 3 طبقات معًا: `Technical SEO` للموقع، `AEO` لبناء صفحات قابلة للاقتباس والإجابة، و`GEO` للظهور داخل Google AI/ChatGPT عبر فهرسة سليمة، كيان واضح، ومحتوى قابل للاستشهاد.
- لا نعتمد أي “AI-only hacks”. الأساس سيكون: فهرسة صحيحة، محتوى موثوق، structured data مطابق للمحتوى، وسماح واضح للزواحف المناسبة. هذا متوافق مع إرشادات Google الحديثة وOpenAI الحالية.

## التغييرات التنفيذية الأساسية
- إعادة هيكلة `app` إلى locale routes:
  - `/ar`, `/en`
  - `/ar/about`, `/en/about`
  - `/ar/projects`, `/en/projects`
  - `/ar/projects/[slug]`, `/en/projects/[slug]`
  - `/ar/blog`, `/en/blog`
  - `/ar/blog/[slug]`, `/en/blog/[slug]`
  - `/ar/contact`, `/en/contact`
- إلغاء الاعتماد على `locale` cookie كوسيلة فهرسة أساسية. يبقى الكوكيز فقط كتفضيل UX، وليس كمصدر تحديد نسخة الصفحة لمحركات البحث.
- إنشاء طبقة SEO مركزية في المشروع تتضمن:
  - `metadataBase` على الدومين النهائي
  - `titleTemplate` منفصل لكل لغة
  - canonical self-reference لكل صفحة
  - `alternates.languages` لكل زوج `ar/en`
  - Open Graph وTwitter cards لكل صفحة
  - robots directives واضحة لكل route
- إضافة ملفات فهرسة قياسية:
  - `app/robots.ts`
  - `app/sitemap.ts`
  - دعم `hreflang` عبر metadata أو sitemap، مع الاكتفاء بطريقة واحدة أساسية لتجنب الازدواج
- إنشاء JSON-LD مركزي ودقيق:
  - `Organization` على الصفحة الرئيسية أو صفحة الشركة
  - `WebSite` للموقع
  - `BreadcrumbList` لكل صفحات التفاصيل
  - `Article` أو `BlogPosting` لكل مقال
  - `WebPage` لصفحات الشركة والمشاريع
- عدم إضافة structured data غير مدعوم أو غير مطابق للنص الظاهر. أي بيانات schema يجب أن تكون موجودة بصريًا على الصفحة نفسها.
- إضافة صفحات/عناصر ثقة إلزامية قبل الإطلاق:
  - سياسة خصوصية
  - الشروط والأحكام
  - صفحة أو قسم “عن الشركة” غني بالكيان، الإدارة، وسائل الاتصال، الموقع، وسنوات الخبرة
- توحيد معلومات الكيان في كل اللغات وكل القنوات:
  - الاسم التجاري
  - الاسم القانوني إن وجد
  - الشعار
  - رقم الهاتف الأساسي
  - عنوان المكتب
  - الروابط الاجتماعية
  - ملف الشركة/البروشور إن كان downloadable
- تفعيل طبقة قياس حقيقية قبل الإطلاق:
  - GA4
  - Google Search Console
  - ربط Search Console مع GA4
  - أحداث تحويل: `whatsapp_click`, `phone_click`, `form_submit`, `brochure_download`, `project_cta_click`

## تغييرات المحتوى والواجهات العامة
- إضافة حقول محتوى جديدة في الـ CMS/مصدر البيانات لكل لغة:
  - `seo_title`
  - `meta_description`
  - `og_title`
  - `og_description`
  - `og_image`
  - `canonical_slug`
  - `indexable`
- إضافة حقول كيان للشركة:
  - `organization_name`
  - `alternate_name`
  - `logo_url`
  - `primary_phone`
  - `address`
  - `same_as[]`
  - `founder_or_chairman`
  - `business_hours` إن وُجدت
- إضافة حقول AEO/GEO للمقالات:
  - `author_name`
  - `reviewed_by` أو `expert_source`
  - `published_at`
  - `updated_at`
  - `faq_blocks[]`
  - `key_takeaways[]`
  - `sources[]`
- إضافة حقول factual للمشاريع:
  - `project_type`
  - `status`
  - `delivery_date`
  - `area_name`
  - `city`
  - `governorate`
  - `amenities[]`
  - `unit_types[]`
  - `payment_plan_summary`
  - `nearby_landmarks[]`
  - `faq_blocks[]`
- اعتماد قوالب محتوى واضحة:
  - الصفحة الرئيسية: كيان الشركة + القيمة + المشاريع + دليل الثقة + CTA مباشر
  - صفحة المشروع: وصف مختصر، حقائق سريعة، مزايا، موقع، حالة المشروع، أسئلة شائعة، CTA واتساب/مكالمة
  - المقال: إجابة مباشرة في أول الشاشة، نقاط مختصرة، شرح، أسئلة شائعة، معلومات الكاتب، مصادر
- سياسة التحرير:
  - العربية هي النسخة المرجعية
  - جميع صفحات التحويل الأساسية bilingual عند الإطلاق
  - المدونة يمكن أن تبدأ بـ Arabic-first مع ترجمة أفضل المقالات التجارية للإنجليزية لاحقًا
- سياسة AEO:
  - كل صفحة مهمة تبدأ بـ summary واضح قابل للاقتباس
  - كل موضوع رئيسي يحتوي FAQ حقيقي
  - كل مقال يربط إلى صفحة مشروع/خدمة/اتصال
  - لا يوجد حشو كلمات مفتاحية أو فقرات عامة بلا معلومة
- سياسة GEO:
  - الحقائق الأساسية قابلة للاستخراج نصيًا، وليست داخل صور فقط
  - المحتوى يذكر الأماكن والكيانات والميزات بأسماء صريحة
  - السماح لـ `OAI-SearchBot` في `robots.txt`
  - عدم ربط الظهور في Google AI بأي schema خاص، لأن Google لا يطلب ذلك

## الحضور الخارجي والإطلاق
- إنشاء/تجهيز Google Search Console على الدومين النهائي قبل الإطلاق العام.
- رفع sitemap بعد الإطلاق مباشرة، ثم فحص الصفحات الأساسية يدويًا:
  - الرئيسية
  - عن الشركة
  - تواصل
  - قائمة المشاريع
  - كل صفحة مشروع
  - قائمة المدونة
  - 3 مقالات أساسية على الأقل
- إنشاء أو تحديث Google Business Profile مع نفس `NAP` الموجود على الموقع، وربطه بصفحة الاتصال والصفحة الرئيسية.
- التأكد أن أزرار الاتصال، الهاتف، واتساب، والعنوان متطابقة نصيًا بين:
  - الموقع
  - Google Business Profile
  - السوشيال
  - أي دليل أعمال رسمي
- تفعيل OpenAI discoverability:
  - السماح لـ `OAI-SearchBot`
  - عدم حظر IP ranges الخاصة به على CDN/WAF
  - الإبقاء على `GPTBot` قرارًا مستقلًا عن الظهور في ChatGPT Search
- خطة الإطلاق:
  - أسبوع 1: route migration + metadata + robots/sitemap + schema
  - أسبوع 2: content model + trust pages + entity completion + analytics
  - أسبوع 3: FAQ/article/project enrichment + GBP/Search Console + validation
  - أسبوع 4: soft launch + crawl checks + indexing requests + KPI baseline

## خطة الاختبار والقبول
- كل صفحة عامة ترجع `200`, قابلة للزحف, وغير محجوبة بـ `robots.txt` أو `noindex`.
- كل صفحة لها:
  - عنوان فريد
  - وصف فريد
  - canonical صحيح
  - `ar/en` alternate صحيح ومتبادل
- لا توجد صفحة تعتمد على الكوكيز لإظهار لغة مختلفة على نفس الـ URL.
- كل schema يطابق النص الظاهر على الصفحة، ويمر على:
  - Rich Results Test حيث ينطبق
  - URL Inspection
- التحقق من:
  - `Organization`
  - `BreadcrumbList`
  - `Article/BlogPosting`
- التحقق من أن `robots.txt` يسمح لـ:
  - `Googlebot`
  - `OAI-SearchBot`
- Lighthouse mobile قبل الإطلاق:
  - SEO = 100
  - Accessibility >= 90
  - Performance >= 85 للصفحات الأساسية
- أحداث التحويل تعمل فعليًا في GA4:
  - واتساب
  - اتصال
  - إرسال نموذج
  - تحميل بروفايل
- قبول الإطلاق لا يتم إلا بعد فحص يدوي لـ:
  - 6 صفحات أساسية بالعربية
  - 6 صفحات أساسية بالإنجليزية
  - 3 صفحات مشروع
  - 3 مقالات

## الافتراضات والقرارات المثبتة
- السوق الأساسي: `Egypt / Arabic-first`.
- نطاق العمل: `الموقع الحالي + الحضور الكامل الخارجي`.
- نوع الإطلاق: `pre-launch`.
- الأولوية التجارية: `WhatsApp + phone qualified leads`.
- بنية اللغة المعتمدة: `separate URLs (/ar, /en)`.
- الدومين النهائي معروف وسيُستخدم في `metadataBase`, canonical, sitemap, Search Console من البداية.
- لن نستخدم FAQ أو Article schema بشكل آلي على أي صفحة لا تحتوي المحتوى الظاهر المطابق.
- لن نعتبر `llms.txt` أو أي ملف AI غير رسمي جزءًا من الـ critical path.
- الخطة تقلل احتمالات إعادة العمل بشكل كبير، لكنها لا تلغي `monthly SEO health review` بعد الإطلاق لأن معايير Google والـ AI تتغير دوريًا.
